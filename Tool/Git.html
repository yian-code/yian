# Git工作区域

工作区工作目录(Working Directory),就是你平时存放项目代码的地方
暂存区Index/Stage,用于临时存放你的改动,事实上它只是一个文件,保存即将提交到文件列表信息
仓库区(或本地仓库)Repository,就是安全存放数据的位置,这里面有你提交到所有版本的数据.其中HEAD指向最新放入仓库的版本
远程仓库git仓库(Remote Directory),托管代码的服务器,可以简单的认为是你项目组中的一台电脑用于远程数据交换

git管理的文件有三种状态

  1. 已修改(modified)
  2. 已暂存(staged)
  3. 已提交(committed)

# 开始使用

  • 创建git仓库
mkdir test
cd test
git init
touch README.md
git add README.md
git commit -m "This is a test"
git remote add origin https://gitee.com/xxx/test.git
git push -u origin master
  • 已有仓库
cd existing_git_repo
git remote add origin https://gitee.com/xxx/test.git
git push -u origin master
  • 配置用户名和用户邮箱
# git全局设置:
git config --global user.name "xxx"
git config --global user.email "xxx@163.com"

使用vs code时需要输入用户名和密码,此时需要在终端中输入以下代码,然后第一次输入用户名和密码后就不需要在输入了

git config --global credential.helper store

# 查看配置

# 查看配置
git config -l

#查看系统config
git config --system --list　
　
#查看当前用户(global)配置
git config --global  --list

# 基本命令

# 查看文件状态   1.工作区 ：红色  2.暂存区：绿色 3.本地仓库：查不到
git status       

# 将文件提交到暂存区  git add . 提交所有文件到暂存区,慎用
git add a.cpp b.h c.cpp

# 将文件从暂存区移出
git rm --cached a.cpp
git reset b.h

# 提交描述信息
git commit -m "submit message"

# 查看提交日志
git log      # ->只能查看一个指针记录    
git reflog   # 查看所有版本的日志
git log --oneline
# fc50797 image major
# f89a1fb image major
# 2da61c5 img submit for yian,breakthrought!

# 版本的前进与后退  git reset --hard [指定版本索引号]
git reset --hard f89a1fb







 
 




 







 

# 分支管理

同时推进多个功能开发,提高开发效率

# 查看、创建、切换分支

git branch -v      # 查看分支
# * master cf154aa 阈值二值化2

git branch tag1    # 创建分支->会以当前分支所在版本的内容来创建分支
git checkout tag1  # 进入该分支
git checkout -b    # 创建并进入该分支

# 合并分支

需要先进入要合并到的目标分支操作,且一定会合并到该分支的最新的版本

合并冲突后需要手动合并,再重新提交

# 将tag1合并到主分支 
git checkout master
git merge tag1 -m 'enter a commit'

# 删除分支

# 删除本地tag1分支
git branch -d tag1

# 删除远程分支
git push origin -d tag1

# 远程仓库

# 查看、添加远程仓库

# 查看你已经配置的远程仓库服务器
git remote 
# origin

# 也可以指定选项 -v,会显示需要读写远程仓库使用的 Git 保存的简写与其对应的 URL
git remote -v
# origin	git@gitee.com:xxx/web.git (fetch)
# origin	git@gitee.com:xxx/web.git (push)

# 添加远程仓库
git remote add <shortname> <url>

# 使用远程仓库

  1. 创建ssh密钥
ssh-keygen -t rsa -C "xxxxx@xxxxx.com"

# 查看密钥
cat ~/.ssh/id_rsa.pub
# ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC6eNtGpNGwstc....

# 测试连接
ssh -T git@gitee.com
  1. 第一次推送
git add .
git commit -m "submit commit"
git remote add origin git@XXXXXX.git
git push -u origin master
  1. 多人开发同一个远程分支时候每次push代码之前需要先pull拉取远程仓库代码再进行push操作
# 克隆远程仓库到本地
git clone ...

# 拉取远程
git pull
# 本地提交到远程仓库
git push

# 多分支开发⭐

我们应该将我们的代码push到另外(通常是你的boss给定)的分支,而不是主分支

这里我们假设BOSS给我在远程仓库上建立了一个供我使用的dev分支,这里我将本地的主分支提交到远程仓库的dev分支:git push origin 本地要上传代码的分支名称:远程你要上传的指定分支名称

# 查看本地和远程的所有分支, 本地分支为绿色,远程分支为红色
git branch --all  

# git branch -v
# git checkout master
# git add 
# git commit -m ""
git push origin master:dev







 

另外如果远程仓库上没有dev这个分支,我们执行git push origin master:dev命令的话,它也会帮我们自动创建这个分支

# 做一个标签⭐

# 查看标签
git tag

# 删除本地标签
git tag -d v1.0.1

git tag -a v1.0.0 -m 'this is first tag'
# 将标签推送到远程仓库
git push origin v1.0.0

  2. 打开.gitignore 文件,添加你要忽略推送的文件,下面是一份忽略清单
.DS_Store 
node_modules

# 开发者个人使用

在.git/info/exclude文件中添加忽略项

# 忽略*.o和*.a文件,不忽略my.o文件
*.[oa]

dbg

# 只忽略dbg文件，不忽略dbg目录
dbg
!dbg/

# 只忽略当前目录下的dbg文件和目录，子目录的dbg不在忽略范围内
/dbg

# 无法更新代码问题

  • 提示没有权限
  1. 可能你真的没有权限
  2. 你的项目没有关联远程仓库
  3. 没有网络或者网络卡
  4. gitee网站的问题
  • 无法git pull 本地仓库修改了代码,需要先commit后才能执行git pull操作

# Github访问问题

linux 的/etc/hosts是配置ip地址和其对应主机名的文件,这里可以记录本机的或其他主机的ip及其对应主机名

sudo gedit /etc/hosts

在网上找了很多资源,好像反应比较强烈的都是通过修改 HOSTS 文件进行加速,手动把cdn和ip地址绑定,我大概的ping了以下几个可用就加进去了,速度飞起.

185.199.108.153 assets-cdn.github.com
192.30.253.118 gist.github.com
185.199.110.153 github.io
199.232.28.133 github.map.fastly.net
199.59.150.39 http://github.global.ssl.fastly.net
52.216.80.48 github-cloud.s3.amazonaws.com
52.216.227.168 github-cloud.s3.amazonaws.com

添加后刷新CDN

sudo /etc/init.d/networking restart
