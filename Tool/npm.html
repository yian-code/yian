NPM的全称是Node Package Manager,是随同NodeJS一起安装的包管理和分发工具,它很方便让JavaScript开发者下载、安装、上传以及管理已经安装的包.

npm官网open in new window

# npm基础

npm -v 

# 查看安装的模块
npm list 
# npm list jquery

# npm 初始化当前目录
npm init  
# 默认初始化
npm init -y
命令描述
-gglobal 的缩写,表示安装到全局目录里
-Ssave 的缩写,表示安装的包将写入package.json里面的dependencies
-Ddev 的缩写,表示将安装的包将写入packege.json里面的devDependencies        
iinstall的缩写,表示安装

# 安装模块

# 安装所有依赖,(会将dependencies中的依赖全部下载)
npm install   
npm i 

# 本地安装:安装模块到默认dependencies  
npm i express
npm install express   

# 安装包信息将加入到dependencies生产依赖
npm install express --save   
npm i express -S 

# 安装包信息将加入到devDependencies开发依赖
npm install express --save-dev 
npm i express -D 

# 安装jquery指定的1.8.3版本
npm install jquery@1.8.3  
npm i jquery@1.8.3  

# 会安装到配置的全局目录下,一般不使用
npm i express -g 
npm install express -global

# 卸载模块

# 卸载模块,但不卸载模块留在package.json中的对应信息
npm uninstall express  
# 卸载全局模块
npm uninstall express -g  
# 卸载模块,同时卸载留在package.json中dependencies下的信息
npm uninstall express --save  
# 卸载模块,同时卸载留在package.json中devDependencies下的信息
npm uninstall express --save-dev

# 查看模块

# 查看安装的模块及依赖
npm ls
# 查看全局安装的模块及依赖
npm ls -g

# 查看当前包安装路径
npm root
# 查看全局的包安装路径
npm root -g

# 查看包依赖关系
npm view moduleName dependencies

# 查看node模块的package.json文件
npm view moduleName

# 查看包依赖的node版本
npm view moduleName engines

# 发布一个npm包时需要查看某个名字是否已经存在
npm search packageName

# 更新模块

# 检查包是否已经过时,会列出已经过时的包,可以及时的进行包更新
npm outdated

 # 更新最新版本的jquery
npm update jquery 

# 更新到指定版本号的jquery
npm update jquery@2.1.0  

# 可以直接更新到最新的版本
npm install jquery@latest

# npm-check

npm i -g npm-check

使用 npm-check 更新项目依赖

npm-check -u

空格切换包是否更新,Control + C​  取消更新,回车就是执行更新
