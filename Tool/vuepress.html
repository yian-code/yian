# 自定义容器

轻松使用vuepress

将常用的Markdown和内置组件等写成了VsCode代码片段,这样就能在记录笔记或博客时轻松使用了,可参考工具/VsCode

tip:提示、warning:警告、danger:危险、details:详情

::: tip
这是一个提示
:::
::: warning
这是一个警告
:::
::: danger
这是一个危险警告
:::
::: details
这是一个详情块,在 IE / Edge 中不生效
:::

你也可以自定义块中的标题:

::: danger STOP
危险区域,禁止通行
:::

# 锚点链接

实现页内跳转或跨页跳转
这是一个锚点链接<a href="#vuepress组件开发">这是一个锚点链接</a>

注意文档规范,当使用页面内锚点链接时,标题不能出现空格等奇怪的东西,不然类名不等于标题名称就无法跳转了,需要自己打开控制台看一看类名就很麻烦

如果文档规范则基本语法就是:<a href="#标题名称">锚点链接名称</a>

# CodeGroupitem

  • title
    • 类型: string
    • 是否必需: true
  • active
    • 类型: boolean
    • 默认值: false
    <CodeGroup>
    <CodeGroupItem title="程序">

    ```bash:no-line-numbers
    yarn
    ```
    </CodeGroupItem>
    <CodeGroupItem title="运行结果">

    ```bash:no-line-numbers
    npm install
    ```

    </CodeGroupItem>
    </CodeGroup>


yarn

npm install

注意

必须在<CodeGroupItem>的开始标签和代码块之间添加一个空行,否则代码块无法被 Markdown 正确解析.

看到这里,你会发现站主是个垃圾,前面只是对vuepress的浅度使用,vuepress可以完成一些更有趣的东西,比如下面的小功能(来自vuepress官网)

你好, Markdown 中的 Vue

实现代码
你好, {{ msg }}

<RedDiv>

_当前计数为: {{ count }}_<input type="text">

</RedDiv>

<button @click="count++">点我！</button>

<script setup>
import { h, ref } from 'vue'

const RedDiv = (_, ctx) => h(
  'div',
  {
    class: 'red-div',
  },
  ctx.slots.default()
)
const msg = 'Markdown 中的 Vue'
const count = ref(0)
</script>

<style>

  color: red;
