# 获取元素

当网页被加载时,浏览器会创建页面的文档对象模型(Document Object Model),通过 HTML DOM,可访问 JavaScript HTML 文档的所有元素

# 1. getElementById()

  • 通过ID属性名获取标签,只能获取到一个唯一的标签(id是唯一的),返回一个匹配特定 ID的元素
    • element是一个Element 对象.如果当前文档中拥有特定ID的元素不存在则返回null.
    • id是大小写敏感的字符串,代表了所要查找的元素的唯一ID
var element = document.getElementById(id);

# 2. querySelector()

返回文档中与指定选择器或选择器组匹配的第一个 Element对象, 如果找不到匹配项,则返回null.

  • 如果匹配到多个重复的标签,只会取第一个标签
var element = document.querySelector(selectors);

# 3. getElementsByClassName()

  • 通过class类型获取标签,返回一个包含了所有指定类名的子元素的类数组对象
  • 获取到的是一个伪数组 也就是说数据是用数组形式保存的,但是无法使用数组对象上提供的相应方法
  • 当没有获取到对应类型的标签时,得到的是一个空数组
    • names 是一个字符串,表示要匹配的类名列表,类名通过空格分隔
var elements = document.getElementsByClassName(names);

获取所有 class 同时包括 'red' 和 'test' 的元素

var elements = document.getElementsByClassName('red test');

# 4. getElementsByTagName()

  • 获取到的是一个伪数组 也就是说数据是用数组形式保存的,但是无法使用数组对象上提供的相应方法
  • 当没有获取到对应类型的标签时,得到的是一个空数组
    • name 是一个代表元素的名称的字符串,特殊字符 " * " 代表了所有元素
var elements = document.getElementsByTagName(name);

# 5. getElementsByName()

  • 根据给定的name(en-US)返回一个在(X)HTML document的节点列表集合
  • 获取到的是一个伪数组 也就是说数据是用数组形式保存的,但是无法使用数组对象上提供的相应方法
  • 当没有获取到对应类型的标签时,得到的是一个空数组
var elements = document.getElementsByName(name);

# 6. querySelectorAll()

返回与指定的选择器组匹配的文档中的元素列表 (使用深度优先的先序遍历文档的节点),返回的对象是节点集合

  • 当没有获取到对应类型的标签时,得到的是一个空集合
var elementList = document.querySelectorAll(selectors);

# 属性操作方法

# setAttribute()

设置指定元素属性.如果属性已经存在则更新该属性,不存在则使用指定的名称和值添加一个新的属性,语法:element.setAttribute(name, value);

var b = document.querySelector("button");

b.setAttribute("name", "helloButton");
b.setAttribute("disabled", "");
# getAttribute()

返回元素上一个指定的属性值,如果指定的属性不存在,则返回 null 或 "" （空字符串）

let attribute = element.getAttribute(attributeName);
# removeAttribute()

从指定元素中删除一个属性

element.removeAttribute(attrName);

# class类操作方法

  • classList  只读属性,返回一个元素的类属性的实时DOMTokenList集合
    • classList.add()  添加一个新的类名
    • classList.remove()  删除一个类名
    • classList.toggle()  切换类名,当类名存在会删除,不存在会添加
    • classList.contains()  判断是否存在某个class类名,存在返回true 不存在在返回false
const box = document.querSelector("box");
box.classList.add('active')
box.classList.remove('boxs')
box.classList.toggle('new-box')

if(box.classList.contains('active')){  //开关控制
	box.classList.remove('active');
}else{
	box.classList.add('active');
}

# DOM/Node属性

属性是什么
nodeName只读,返回当前节点的节点名称,得到的是一个大写的标签名(toLowerCase转小写)
nodeType返回该节点类型(文本节点3、属性节点2、标签节点1)
parentNode表示获取到的该节点的父节点
childNodes表示获取到该元素的所有子节点,以数组形式保存
firstElementChild(lastElementChild)获取第一个(最后一个)子元素节点
firstChild获取该元素下第一个节点,不一定是元素节点,有可能是文本、注释等其他节点
nextElementSibling(previousElementSibling)获取该元素的下一个(上一个)兄弟元素
clientWidth获取元素内容的宽度,也就是内容 + 内边距 的宽度
clientHeight获取元素内容的高度,包含内容 + 内边距
offsetWidth获取元素的实际占位宽度 包含 内容 + 内边距 + 边框
offsetHeight获取元素的实际占位高度 包含 内容 + 内边距 + 边框
offsetLeft默认情况下获取元素与浏览器左侧的距离,如果父级元素有定位,则表示获取与有定位的父级元素左侧的距离
offsetTop默认情况下获取元素与浏览器上方的距离,如果父级元素有定位,则表示获取与有定位的父级元素上方的距离
scrollTop元素内部滚动条滚动的距离
document.documentElement.scrollTop表示获取浏览器默认的滚动条滚动的高度 整个HTML有doctype声明时获取的方式
document.body.scrollTop表示获取浏览器默认的滚动条滚动的高度 整个HTML没有doctype声明时获取的方式

# remove

把对象从它所属的 DOM 树中删除

node.remove();
# removeChild

Node.removeChild() 方法从DOM中删除一个子节点.返回删除的节点

  • child  是要移除的那个子节点.
  • node  是child的父节点.
  • oldChild 保存对删除的子节点的引用. oldChild === child.
let oldChild = node.removeChild(child);
//OR
element.removeChild(child);
# replaceChild

Node.replaceChild() 方法用指定的节点替换当前节点的一个子节点,并返回被替换掉的节点

parentNode.replaceChild(newChild, oldChild);
  • newChild: 用来替换 oldChild 的新节点,如果该节点已经存在于 DOM 树中,则它首先会被从原始位置删除
  • oldChild: 被替换掉的原始节点
# cloneNode

Node.cloneNode() 方法返回调用该方法的节点的一个副本

var dupNode = node.cloneNode(deep);
  • node: 将要被克隆的节点
  • dupNode: 克隆生成的副本节点
  • deep:  可选.是否采用深度克隆,如果为true,则该节点的所有后代节点也都会被克隆,如果为false,则只克隆该节点本身.

# DOM2级事件处理

  • 绑定事件监听器 语法:标签.addEventListener('事件名',函数名)
  • 移除事件监听器 语法:标签.removeEventListener("事件名",函数名).前提条件,事件处理函数必须是一个有名字的函数,不能是匿名函数.

DOM0级和DOM2级事件处理之间的不同:

  1. DOM0级无法同时绑定多个相同事件,后绑定的会覆盖先绑定的,DOM2级可以同时绑定相同事件,不会覆盖
  2. DOM0级无法控制事件的响应阶段,DOM2级可以设置第三个参数,用来控制事件流的响应阶段,默认false表示冒泡阶段,true表示捕获阶段

# 事件委托 事件代理

主要应用事件流的响应,以及事件对象event中的 e.target 来实现 以下是一个按钮事件代理案例,这样原本需要为两个按钮分别绑定的触发函数就变成了一个函数

<li> <button class="btn1">按钮一</button> <button class="btn2">按钮二</button> </li>
<script>
	var li = document.querySelector('li');

	li.onclick = function(e){
		var b = e.target; //当按钮一点击时: <button class="btn1">
		if(b.className == "btn1"){
			console.log("按钮一点击")
		}else if(b.className == "btn2"){
			console.log("按钮二点击")
		}else{
			console.log("点击了大盒子")
		}
	}
</script>

# 事件流

捕获   →  冒泡

事件流的执行顺序是从外向内层层捕获,捕获到最终触发事件的标签后,再由内向外层层冒泡,这个执行的过程就是事件流,目前所有的新式浏览器事件响应状态都在冒泡阶段触发

  • 阻止事件流 event.stopPropagation()
maxBox.onclick = function(event){
	var event = event || window.event;	// IE没有event,需要这样使用,现在一般不需要了
	if(event.stopPropagation){	//兼容性语法
		event.stopPropagation(); //通用浏览器阻止事件流
	}else{
		event.cancelBubble = true; //IE 6 7 8 阻止事件流
	}
	console.log("maxbox is clicked")
}

添加在事件函数的最起始位置 可以阻止浏览器默认事件

  • return false

添加在事件函数的最末尾 可以阻止浏览器默认事件

阻止事件流案例:自定义浏览器右键菜单

var contextMenu = document.getElementById("context-menu");
window.oncontextmenu = function(event){
	event.preventDefault(); //禁用掉默认的后可以当做单击事件使用
	console.log(event)
	var x =event.clientX || event.pageX;
	var y = event.clientY || event.pageY;
	contextMenu.style.left = x + "px";
	contextMenu.style.top = y + "px";
	contextMenu.style.display = "block";
	// return false;
}
// 点击任意位置隐藏右键盒子
document.onclick = function(){
	contextMenu.style.display = "none";
}

#context-menu{
	width: 100px;
	height: 200px;
	border: 1px solid #ccc;
	background: #333;
	position: fixed;
	top: 0;
	left: 0;
	display: none;
}
