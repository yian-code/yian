# var、const、let

# var关键字

var语句用于声明一个函数范围或全局范围的变量,无论在何处声明变量,都会在执行任何代码之前进行处理.这被称为变量提升

显式与隐式变量

  • 显式创建:带var的创建方式
  • 隐式创建:不带var的创建方式

# const关键字

常量是块级范围的,非常类似用let语句定义的变量.但常量的值是无法(通过重新赋值)改变的,也不能被重新声明,本人理解应该是与C++等语言中作用是一样的

  • 一旦声明必须赋值,不能使用null占位
  • 如果值是基本数据类型,则值无法被改变
  • 如果声明的是引用数据类型,则地址无法改变,但值可以修改
{
    const obj = {
        age: 20
    }
    obj.age = 30;  // 这是允许的
}

一个常量不能和它所在作用域内的其他变量或函数拥有相同的名称

# let关键字

let声明一个块级作用域的局部变量,并可以初始化为一个值.

let value = 1;

let与var的区别:

  • let允许你声明一个作用域被限制在块作用域中的变量、语句或者表达式.暂时性死区
  • var声明的变量作用域是全局或者整个函数块的.不存在死区
  • var和let的另一个重要区别,let声明的变量不会在作用域中被提升,也就是必须在使用之前就创建,否则会报错
function fn(){
    console.log(a);
    var a = 10;
}
fn(); // undefined 

function fn(){
    console.log(a);
    let a = 40;
}
fn();
// Uncaught ReferenceError: Cannot access 'a' before initialization
  • 在全局作用域中来看,let不会在顶层对象上创建属性:
var a = 'yian';
let b = 'yian';

console.log(this.a); // yian
console.log(this.b); // undefined

# Class基本语法

基本上,ES6 的class可以看作只是一个语法糖

class Person{
    sex = 1  // 公共属性可以直接写
    static id = "001007007010"; // 静态成员
    constructor(name, age) { // 传参:constructor 
        this.name = name;
        this.age  = age;
    }
    say(){  // 类成员方法,直接绑定在原型上
        console.log("class say");
    };
}
let user = new Person("lLL",18);  // 

console.log(user);
// Person {sex: 1, name: 'lLL', age: 18}
// age:18
// name:'lLL'
// sex:1
// [[Prototype]]:Object 
// constructor:.....  
// say:ƒ say(){console.log("class say");}
// [[Prototype]]:Object

console.log(user.name);
// lLL
user.say()
// class say

console.log(user.id);
// undefined
console.log(Person.id)
// 001007007010
  • 静态成员:只能被当前类本身所访问到,需要在对应属性或方法前面加上一个static关键字
  • 动态成员:当前类本身无法访问,只能被所有实例访问

# 箭头函数⭐

箭头函数表达式的语法比函数表达式更简洁,并且没有自己的this,arguments,super或new.target.箭头函数表达式更适用于那些本来需要匿名函数的地方,并且它不能用作构造函数.

var fn = (a, b) => {
    console.log(a + b);
}
fn(10, 20);

var fn = function(a, b){
    console.log(a + b);
}
fn(10, 20);

当函数的参数只有一个时小括号也可以简写

var fn2 = param => { console.log(param); }

当函数语句只有一条时,可以再次简写(不要return)

var fn3 = param => param.toString()

var fn3 = function(param) {
    return param.toString();
}

在箭头函数中没有自己的this指向, 上级作用域的this指向谁,箭头函数中的this就指向谁

var obj = {
    fn: ()=>{
        console.log(this);
    },
    fn1: function(){
        console.log(this);
    }
}
obj.fn();
// Window
obj.fn1();
// Object { fn: fn(), fn1: fn1() }

# 剩余运算符(扩展运算符)⭐

# 剩余运算符

一般是在函数的最后一个形参使用,给最后一个形参添加...,表示将剩余所有实参交给最后一个形参以数组形式保存

function fn(a, b, ...c) {
    console.log('a:',a);
    console.log('b:',b);
    console.log('c:',c);
}
fn(1, 2, 3, 4, 5, 6, 7, 8, 9, 0);

// a: 1 
// b: 2 
// c: Array(8) [ 3, 4, 5, 6, 7, 8, 9, 0 ]
 









# 扩展运算符

主要是为了展开一个数组

var arr = [1, 3, 2];

function fn1(a, b, c) {
    console.log(a + b + c);
}

fn1(...arr);
// 6

var arr1 = [1, 2, 3];
var arr2 = [4, 5, 6];

var arr3 = arr1.concat(arr2);
// Array(6) [ 1, 2, 3, 4, 5, 6 ]

var arr4 = [...arr1, ...arr2]
console.log(arr4);
// Array(6) [ 1, 2, 3, 4, 5, 6 ]

# 解构赋值⭐

ES6中允许按照一定的模式从数组和对象中提取值.

解构赋值语法是一种 Javascript 表达式,通过解构赋值,可以将属性/值从对象/数组中取出,赋值给其他变量.

# 解构对象

var personSystemInfoUser = {
    userName: 'yian',
    age: 20,
    sex: '1'
}
// var userName = personSystemInfoUser.userName;
// var age = personSystemInfoUser.age;
// var sex = personSystemInfoUser.sex;

var { userName, age, sex } = personSystemInfoUser;
console.log(userName, age, sex);
// yian 20 1

变量名必须和属性名一样,且在声明时候必须同时赋值

var { Name, age, sex } = personSystemInfoUser;
console.log(userName, age, sex);
// Uncaught ReferenceError: userName is not defined

var { userName, age, sex };
{ userName, age, sex } = personSystemInfoUser;
// Uncaught SyntaxError: missing = in destructuring declaration

# 解构数组

变量名可以随意设置,但是解构的内容必须从数组中依次拿取.

var personUserInfo = [1, 2, 3];
var [a, b, c] = personUserInfo;
console.log(a, b, c);
// 1 2 3

var [a, ...c] = personUserInfo;
console.log(a, c);
// 1 Array [ 2, 3 ]

 



 


不要试图给解构后的变量赋值以改变解构的数组或对象,因为解构是直接等同于使用,而不是简单的赋与地址

var arr = [1, 2, 3]

var [a, b, c] = arr; 
/**  =  
  **** var a = arr[0];
  **** var b = arr[1];
  **** var c = arr[2];
**/
console.log('a =', a);
// a = 1

a = 9;
console.log('a =', a);
// a = 9
console.log(arr);
// Array(3) [ 1, 2, 3 ]

# 对象函数简写

var obj = {
    userName,
    age,
    fn: function(){ console.log('函数fn调用') }
}
obj.fn();
// 函数fn调用

可以直接简写为：

var obj = {
    userName,
    age,
    fn(){ console.log('函数fn调用') }
}
obj.fn();
// 函数fn调用

如果函数写在外面也可以简化

var fn = function(){ console.log('函数fn调用') };
var obj = {
    userName,
    age,
    fn()
}
obj.fn();
// 函数fn调用

# 函数参数的默认值

以上例子中我们给的参数少了一个,函数会将缺少的参数默认为undefined.所以返回结果为NaN

function fn(a, b, c){
    console.log(a + b + c);
}
fn(1, 2)
// 1 + 2 + undefined = NaN

在ES6后,ES6 允许为函数的参数设置默认值,即直接写在参数定义的后面

function fn(a, b, c = 0){
    console.log(a + b + c);
}
fn(1, 2)
// 1 + 2 + 0 = 3

再来看另外一个例子

var userObj = {
    name: 'yian',
    age: 20,
    sex:'男',
    _id:'aaa'
}

function logObj(name, age, sex, _id) {
    console.log(name, age, sex, _id);  
}
logObj(userObj.name, userObj.age, userObj.sex)
// yian 20 男 undefined

使用logObj函数的时候传递的参数实在是太麻烦了,你可以使用以下方法来简化

function logObj( {name, age, sex, _id = 'bbb'} ) {
    console.log(name, age, sex, _id);  
}
logObj(userObj);
// yian 20 男 bbb
