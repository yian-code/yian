# var、const、let

ES5只有两种声明变量的方式:var和function,ES6除了添加了let和const,还添加了import与class命令,所以ES6一共有六种声明变量的方法

编程风格

  • 建议不再使用var命令,而是使用let命令取代.
  • let与const之间优先使用const,尤其是在全局环境,不应该设置变量,只应设置常量,有利于提高程序运行效率
  • 所有的函数都应该设置为常量

# const关键字

常量是块级范围的,非常类似用let语句定义的变量.但常量的值是无法(通过重新赋值)改变的,也不能被重新声明,本人理解应该是与C++等语言中作用是一样的

  • 一旦声明必须赋值,不能使用null占位
  • 如果值是基本数据类型,则值无法被改变
  • 如果声明的是引用数据类型,则地址无法改变,但值可以修改
{
    const obj = {
        age: 20
    }
    obj.age = 30;  // 这是允许的
}

一个常量不能和它所在作用域内的其他变量或函数拥有相同的名称

# var关键字与let关键字

  • var语句用于声明一个函数范围或全局范围的变量,无论在何处声明变量,都会在执行任何代码之前进行处理.这被称为变量提升

显式与隐式变量

  • 显式创建:带var的创建方式
  • 隐式创建:不带var的创建方式
  • let声明一个块级作用域的局部变量,并可以初始化为一个值.let value = 1;

let与var的区别:

  • let允许你声明一个作用域被限制在块作用域中的变量、语句或者表达式.暂时性死区
  • var声明的变量作用域是全局或者整个函数块的.不存在死区
for(var a = 1; a < 5; a++){
    setTimeout(() => {
        console.log('a:', a);
    }, 0);
    console.log(a);
}
// a: 5
// a: 5
// a: 5
// a: 5

for(let a = 1; a < 5; a++){
    setTimeout(() => {
        console.log('a:', a);
    }, 0);
    console.log(a);
}
// a: 1
// a: 2
// a: 3
// a: 4
  • var和let的另一个重要区别,let声明的变量不会在作用域中被提升,也就是必须在使用之前就创建,否则会报错
function fn(){
    console.log(a);
    var a = 10;
}
fn(); // undefined 

function fn(){
    console.log(a);
    let a = 40;
}
fn();
// Uncaught ReferenceError: Cannot access 'a' before initialization
  • 在全局作用域中来看,let不会在顶层对象上创建属性:
var a = 'yian';
let b = 'yian';

console.log(this.a); // yian
console.log(this.b); // undefined

# 解构赋值

ES6中允许按照一定的模式从数组和对象中提取值.解构赋值语法是一种 Javascript 表达式,通过解构赋值,可以将属性/值从对象/数组中取出,赋值给其他变量.

  • 对象解构赋值
  • 数组解构赋值
  • 字符串解构赋值
  • 函数参数的解构赋值

# 对象解构赋值

var personSystemInfoUser = {userName: 'yian', age: 20, sex: '1'}
// var userName = personSystemInfoUser.userName;
// var age = personSystemInfoUser.age;
// var sex = personSystemInfoUser.sex;

var { userName, age, sex } = personSystemInfoUser;
console.log(userName, age, sex);
// yian 20 1
  • 变量名必须和属性名一样,且在声明时候必须同时赋值,如果解构失败,变量的值等于undefined.
var { Name, age, sex } = personSystemInfoUser;
console.log(userName, age, sex);
// Uncaught ReferenceError: userName is not defined

var { userName, age, sex };
{ userName, age, sex } = personSystemInfoUser;
// Uncaught SyntaxError: missing = in destructuring declaration
  • 当变量名与属性名不一致时,必须使用以下写法
var objUser = { userName: 'yian', sex: 1, id:'212121212' }

var { userName:name, sex:sex, id:_id } = objUser;

console.log('name =', name, 'sex =', sex, '_id =', _id);
// name = yian sex = 1 _id = 212121212


 



  • 这也说明了对象的解构赋值机制是先找到同名的属性,然后再赋值给对应的变量,真正被赋值的是变量而不是属性

  • 对象的解构赋值也可以很方便的将现有对象的方法赋到某个变量上

let { log, sin, cos } = Math;

const { log } = console;
log('hello yian') // hello yian
  • 解构也可用于嵌套结构的对象
let obj = {
  p: [
    'Hello',
    { y: 'yian' }
  ]
};

let { p: [x, { y }] } = obj;
x // "Hello"
y // "yian"
  • 也可以像数组解构一样赋与默认值.默认值生效的条件是属性值严格等于undefined
  • 如果要使用已经声明好的变量用于解构赋值,需要注意以下情况
var personSystemInfoUser = {userName: 'yian', age: 20, sex: '1'}

// 错误的做法,->大括号在行首会被解析成代码块
var userName, age, sex ;
{ userName, age, sex } = personSystemInfoUser;

// 正确的
var userName, age, sex ;
({ userName, age, sex } = personSystemInfoUser);
console.log(userName, age, sex);

解构赋值的拷贝是浅拷贝.解构赋值拷贝的是这个值的引用,而不是这个值的副本

# 数组解构赋值

变量名可以随意设置,但是解构的内容必须从数组中依次拿取.

var [a, b, c] = [1, 2, 3];
console.log(a, b, c);
// 1 2 3

var [a, ...c] = [1, 2, 3];
console.log(a, c);
// 1 Array [ 2, 3 ]

 



 

只要左右两边的模式相同,就可以进行合法赋值

let [A, [[B], C]] = [10, [[20], 30]];

console.log('A=', A, 'B=', B, 'C=', C);
// A= 10 B= 20 C= 30

解构可以赋默认值

let [name = 'yian'] = ['易安'];
// let [name = 'yian'] = [];
console.log('name=', name);
// name= 易安
// name= yian

不要试图给解构后的变量赋值以改变解构的数组或对象,因为解构是直接等同于使用,而不是简单的赋与地址

var arr = [1, 2, 3]

var [a, b, c] = arr; 
/**  =  
  **** var a = arr[0];
  **** var b = arr[1];
  **** var c = arr[2];
**/
console.log('a =', a);
// a = 1

a = 9;
console.log('a =', a);
// a = 9
console.log(arr);
// Array(3) [ 1, 2, 3 ]

# 字符串解构赋值

字符串可以进行解构赋值,因为字符串可以被转换为类似于数组的对象

const [a, b, c, d] = 'yian';
a // y
b // i 
c // a 
d // n

还可以对字符串的length属性进行解构赋值

let { length : len } = 'yian';
len // 4

# 函数参数的解构赋值

var userObj = {
    name: 'yian',
    age: 20,
    sex:'男',
    _id:'aaa'
}

function logObj(name, age, sex, _id) {
    console.log(name, age, sex, _id);  
}
logObj(userObj.name, userObj.age, userObj.sex)
// yian 20 男 undefined

使用logObj函数的时候传递的参数实在是太麻烦了,你可以使用以下方法来简化.

function logObj( {name, age, sex, _id = 'bbb'} ) {
    console.log(name, age, sex, _id);  
}
logObj(userObj);
// yian 20 男 bbb

函数参数的解构也可以赋默认值

function add([a = 0, b = 0]){
    return a + b;
}
add([2, 5]) // 7

# 字符串新增方法

简单的学了几个,具体的还是需要看文档

  • String.fromCodePoint(): 用于从 Unicode 码点返回对应字符
  • String.raw(): 返回一个斜杠都被转义的字符串,大多用于模板字符串的处理
console.log( `Hi\n${2+3}!` ) 
// Hi
// 5!
console.log( String.raw`Hi\n${2+3}!` )  
// 'Hi\n5!'
  • codePointAt(): 正确处理 4 个字节储存的字符,返回一个字符的码点.
  • normalize() 之前学到的indexOf方法可以确定一个字符串是否包含在另一个字符串中,ES6新增了以下三个方法
  • includes: 表示是否找到该字符串,返回一个布尔值
  • startsWith: 表示参数字符串是不是在原字符串头部,返回一个布尔值
  • endsWith: 表示参数字符串是不是在原字符串尾部,返回一个布尔值

三个方法都支持第二个参数,表示搜索开始位置

let str = 'yianjun';

str.includes('yian')    // true
str.startsWith('y')     // true
str.endsWith('n')       // true

str.includes('yian', 2)    // false
str.startsWith('y', 0)     // true
str.endsWith('n', 4)       // true
  • 实例方法:repeat(),返回一个新的字符串,表示将原字符串重复N次
'an'.repeat(3) // ananan
  1. 参数如果是小数则会被向下取整'na'.repeat(2.9) // "nana"
  2. 参数在0到-1之间等同于零,返回空字符串
  3. 参数NaN等同于0
  4. 参数为负数或者Infinity会报错
  5. 参数是字符串会先转换为数字
'an'.repeat('yi') // ''
'an'.repeat('3')  // ananan
  • 实例方法padStart(),padEnd(): 用于补全字符串,如果字符串不够长则会在头部或尾部进行补全
  1. 第一个参数为补全后的最大长度,第二个是用来补全的字符串
'x'.padStart(6, 'ab') // ababax
  1. 原字符串长度大于等于最大长度返回原字符串
'xxx'.padStart(2, 'ab') // xxx
  1. 补全字符串加上原字符串后长度大于最大长度,则会将补全字符串截去超出的长度,如例一.
  2. 省略第二个参数,则默认使用空格补全

常见用途是为数值补全指定位数.下面代码生成 10 位的数值字符串

'1'.padStart(10, '0') // "0000000001"

另一个用途是提示字符串格式

'12'.padStart(10, 'YYYY-MM-DD') // "YYYY-MM-12"
  • trimStart(),trimEnd()消除头部或尾部空格
  • matchAll
  • replaceAll(): 见名知意,替换所有,用法与replace()相同
'aaabbbccc'.replaceAll('b', 'B')
// aaaBBBccc
  • at()方法接受一个整数作为参数,返回参数指定位置的字符,支持负索引(即倒数的位置)
const str = 'hello';
str.at(1) // "e"
str.at(-1) // "o"

如果参数位置超过字符串范围则返回undefined

  1. 二进制八进制表示法

  1. Number.isInteger(),判断一个数值是否为整数,如果对精度要求较高不建议使用
  2. Number.EPSILON,表示1与大于一的最小浮点数之差

  4. Math对象的扩展

const a = 2123131313n;
const b = 2343545456n;
console.log( a * b );  // 4975654741072463728n

const a = 2123131313;
const b = 2343545456;
console.log( a * b );
// 4975654741072464000
  • BigInt类型与普通类型的值不相等42 === 42n //false
  • BigInt可以使用负号但是不能使用正号因为会与 asm.js 冲突
  • BigInt函数:与Number转换规则基本一致,将其他类型值转换为BigInt类型

# 函数参数的默认值

以上例子中我们给的参数少了一个,函数会将缺少的参数默认为undefined.所以返回结果为NaN

function fn(a, b, c){
    console.log(a + b + c);
}
fn(1, 2)
// 1 + 2 + undefined = NaN

在ES6后,ES6 允许为函数的参数设置默认值,即直接写在参数定义的后面

function fn(a, b, c = 0){
    console.log(a + b + c);
}
fn(1, 2)
// 1 + 2 + 0 = 3

# 箭头函数

箭头函数表达式的语法比函数表达式更简洁,并且没有自己的this,arguments,super或new.target.箭头函数表达式更适用于那些本来需要匿名函数的地方,并且它不能用作构造函数.

var fn = (a, b) => {
    console.log(a + b);
}
fn(10, 20);

var fn = function(a, b){
    console.log(a + b);
}
fn(10, 20);

当函数的参数只有一个时小括号也可以简写

var fn2 = param => { console.log(param); }

当函数语句只有一条时,可以再次简写(不要return)

var fn3 = param => param.toString()

var fn3 = function(param) {
    return param.toString();
}

在箭头函数中没有自己的this指向, 上级作用域的this指向谁,箭头函数中的this就指向谁

var obj = {
    fn: ()=>{
        console.log(this);
    },
    fn1: function(){
        console.log(this);
    }
}
obj.fn();
// Window
obj.fn1();
// Object { fn: fn(), fn1: fn1() }

# 扩展运算符

这里记录一下数组扩展运算符的应用

  • 复制数组
const a1 = [1, 2, 3];
// 1
const a2 = [...a1];
// 2
const [...a2] = a1;
  • 合并数组->都是浅拷贝
const a1 = ['a'];
const a2 = ['b', 'c']
// ES5 
a1.concat(a2);
// [ 'a', 'b', 'c' ]
// ES6 
[...a1, ...a2];
  • 与解构赋值结合
a = list[0];
// ES5 
newArr = list.slice(1);
// ES6 
[a, ...newArr] = list;

如果将扩展运算符用于数组赋值,则只能放在参数的最后一位const [a, b, ...c] = [1, 2, 3, 4, 5]

  • 字符串:扩赞运算符还能将字符串转换为真正的数组
[...'yian']
// ['y', 'i', 'a', 'n']

还可以在使用函数的时候来进行传参,将数组转化为函数参数序列

var arr = [1, 3, 2];
function fn1(a, b, c) {
    console.log(a + b + c);
}

fn1(...arr);
// 6

# Array.from()

用于将类数组和可遍历的对象(包括Set和Map)转为真正的数组.常见的类似数组的对象是 DOM 操作返回的 NodeList 集合,以及函数内部的arguments对象,都可以转为数组

// NodeList 对象
let btns = document.querySelectorAll('button');
Array.from(btns).filter(item => {
  return item.textContent.length > 100;
});

// arguments 对象
function foo() {
  var args = Array.from(arguments);
  // ...
}

# Array.of()

将一组值转换为数组,没有参数则返回空数组

Array.of(1, 2, 3) // [1, 2, 3]
Array.of()        // []

# find()和findIndex()

  • find()返回第一个符合条件的数组成员,参数为一个回调函数,所有成员都执行一次该函数,返回符合条件的,未找到则返回undefined.
[1, 3, 4, 5, 7].find( (item) => item > 4)  // 5

find方法的回调函数可以接受三个参数,依次为当前的值,当前的位置和原数组.

  • findIndex()与find类似,返回第一个符合条件的位置
[1, 3, 4, 5, 7].findIndex( (item) => item > 4)  // 3

# fill()

使用给定值填充数组,fill方法还可以接受第二个和第三个参数,用于指定填充的起始位置和结束位置

['a', 'b', '3'].fill(6)   // [ 6, 6, 6 ]

[1, 2, 3, 4, 5, 6, 7, 8, 9, 0].fill('a', 3, 8);
// [1, 2, 3, 'a', 'a', 'a', 'a', 'a', 9, 0]

# entries(),keys(),values()

三种方法都用于遍历数组,它们都返回一个遍历器对象,可以用for...of遍历

  • keys():对键名遍历
  • values():对键值遍历
  • entires():对键值对遍历
let arr = ['a', 'b', 'c'];

for (let index of arr.keys()) { console.log(index) }
// 0
// 1
// 2
for (let val of arr.values()) { console.log(val) }
// a
// b
// c
for (let [index, val] of arr.entries()) { console.log([index, val]) }
// [ 0, 'a' ]
// [ 1, 'b' ]
// [ 2, 'c' ]

# flat()

数组的成员有时还是数组,Array.prototype.flat()用于将嵌套的数组拉平,变成一维的数组.该方法返回一个新数组,对原数据没有影响

[1, 2, [3, 4, 5]].flat()     // [1, 2, 3, 4, 5]

flat()默认拉平一层,你可以传一个整数参数表示想要拉平的维度

[1, 2, [3, [4, 5]]].flat(2)  // [1, 2, 3, 4, 5]

如果将N维数组转为一维数组,则参数给Infinity

[1, [2, [3]]].flat(Infinity) // [1, 2, 3]

原数组有空位则会跳过空位

[1, 2, , 4, 5].flat() // [1, 2, 4, 5]
# flatMap()

相当于对原数组每个成员执行一次map()函数再执行flat()方法展开,该方法返回一个数组,不改变原数组

[1, 2, 3, 4].flatMap(x => x * 2) // [ 2, 4, 6, 8 ]

flatMap()只能展开一层数组

[1, 2, 3, 4].flatMap( x => [[x * 2]] ) // [ [ 2 ], [ 4 ], [ 6 ], [ 8 ] ]

# at()

js中的数组不支持负索引,通常表述数组最后一项只能使用arr[arr.length - 1].at()接收整数参数来返回对应成员,支持负索引,超出数组范围返回undefined,不仅可用于数组,也可用于字符串和类型数组(TypedArray)

const arr = [1, 3, 21, 44];
arr.at(2)                   // 21
arr.at(-2)                  // 21
arr.at(4)                   // undefined

# Symbol

Symbolopen in new window

  • 解决属性名冲突问题,它的值通过Symbol函数生成,相同Symbol函数创建的值是唯一的.

# 严格模式

ES6 的模块自动采用严格模式,不管你有没有在模块头部加上use strict;

  • 变量必须声明后再使用
  • 函数的参数不能有同名属性,否则报错
  • 不能使用with语句
  • 不能对只读属性赋值,否则报错
  • 不能使用前缀 0 表示八进制数,否则报错
  • 不能删除不可删除的属性,否则报错
  • 不能删除变量delete prop,会报错,只能删除属性delete global[prop]
  • eval不会在它的外层作用域引入变量
  • eval和arguments不能被重新赋值
  • arguments不会自动反映函数参数的变化
  • 不能使用arguments.callee
  • 不能使用arguments.caller
  • 禁止this指向全局对象
  • 不能使用fn.caller和fn.arguments获取函数调用的堆栈
  • 增加了保留字(比如protected、static和interface)

ES6模块中顶层的this指向undefined,即不应该在顶层使用this

# 编程风格(代码规范)

# 块级作用域

  • 建议不再使用var命令,而是使用let命令取代.
  • let与const之间优先使用const,尤其是在全局环境,不应该设置变量,只应设置常量,有利于提高程序运行效率
  • 所有的函数都应该设置为常量

# 字符串

静态字符串一律使用单引号或反引号,不使用双引号.动态字符串使用反引号

# 解构赋值

  • 使用数组成员对变量赋值时,优先使用解构赋值
const arr = [1, 2, 3, 4];

// bad 
const a = arr[0];
const b = arr[1];

// good 
const [a, b] = arr;
  • 函数的参数如果是对象的成员,优先使用解构赋值
// bad
function userName(user) {
  const firstName = user.firstName;
  const lastName = user.lastName;
}

// good
function userName(obj) {
  const { firstName, lastName } = obj;
}

// best
function userName({ firstName, lastName }) {
}
  • 如果函数返回多个值,优先使用对象的解构赋值,而不是数组的解构赋值.这样便于以后添加返回值,以及更改返回值的顺序
// bad
function processInput(input) {
  return [left, right, top, bottom];
}

// good
function processInput(input) {
  return { left, right, top, bottom };
}

const { left, right } = processInput(input);

# 对象

  • 单行定义对象最后一个成员不以逗号结尾,多行定义对象,最后一个成员以逗号结尾
// bad
const a = { a1: 2, a2: 4, }
const b = {
    a1: 2,
    a2: 4
};
// good
const a = { a1: 2, a2: 4 }
const b = {
    a1: 2,
    a2: 4,
};
  • 对象尽量静态化(const),一旦定义就不得随意添加属性,添加属性使用Object.assign()方法
  • 如果对象的属性名是动态的,可以在创造对象的时候,使用属性表达式定义
  • 对象的属性和方法尽量使用简洁表达法
