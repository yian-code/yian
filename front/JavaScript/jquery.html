JS中的load事件个jq中的ready事件区别

  1. load 一般只给window添加,且只能添加一次
  2. ready可以给任意标签添加,且可以重复添加
  3. ready表示当标签加载完就会执行函数,不需要等里面的内容加载完毕
  4. load表示页面所有标签和内容全部加载完执行函数

jQ中的事件都是DOM2级事件,可重复绑定,所以建议每次先解绑,再绑定事件

  • off(事件名)用于解除绑定事件
let i = 1;
function add () {
    var jQli = $('<li></li>');
    $('ul').append(jQli);
    $('li').off('click').click( function () {
        let _that = this;
        setInterval(() => {
            $(_that).html(i++);
        }, 1000);

    })
}




 







  • on方法可以绑定所有已有事件,还可以绑定自定义事件,需要使用trigger方法触发
  • trigger方法用于触发事件
// on绑定自定义事件
$(document).on('my_click', function (event, a, b) {
    console.log(a + b);
})
// trigger 触发事件
$(document).trigger('my_click', [10, 20]);
// 30
  • one为每一个匹配元素的特定事件（像click）绑定一个一次性的事件处理函数
$('button').one('click', function () {
    console.log(1);
})
  • hover(fn, fn)该方法是mouseover和mouseout两个事件的结合体,第一个参数表示鼠标进入执行,第二个表示鼠标离开执行
 $('div').hover( function () {
    $(this).css('background', 'red')
}, function () {
    $(this).css('background', 'blue')
})
