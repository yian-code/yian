# 作用域和作用域链

  • 全局作用域:全局变量
  • 局部变量 :局部变量(函数私有变量) 只能通过var来创建
  • 作用域链:当某个局部作用域内没有找该变量,那么就会向上级作用域进行查找,如果上级也没有,则继续向上级查找,一直找到全局作用域为止,这种层层查找的过程被称为作用域链

# 闭包函数

当我们需要在一个函数中去访问另一个函数的私有变量时,那么具有该私有变量的函数就是一个闭包函数.通常使用只有一个方法的对象的地方,都可以使用闭包.

缺点:容易造成内存泄露
优点:避免全局变量污染

function init() {
  var name = "YiAn";    // init中的局部变量
  function dspName() {  // dspName是内部函数,一个闭包
    console.log("MyName:", name); // 使用了父函数中声明的变量
  }
  dspName();
}
init();
// MyName: YiAn

# 构造函数

在JavaScript中,用new关键字来调用的函数,称为构造函数,构造函数首字母一般大写

function Person(name, age, sex) {
    this.name = name;
    this.age= age;
    this.sex= sex;
}

var user1 = new Person('yian', 18, 1);
var user2 = new Person('YiAn', 20, 1);

Person构造函数执行过程:

  1. 当以new关键字调用会创建出一个新的工作空间,标记为Person的实例
  2. 将空对象的原型绑定为new操作符后面的构造函数的原型
  3. 函数体内部的this指向该内存,并执行构造函数的函数体,创建相关属性和方法
  4. 判断执行结果是否返回了新的对象,如果有返回新对象,就是用返回的新对象,没有的话,返回原来创建的对象(obj)

# 深浅拷贝 理解JS中的堆栈内存

堆栈内存:

  • 值类型的栈内存:主要针对的 Number Boolean String 三种数据,直接存在栈(stack)内存,占据的空间小,大小固定,被频繁使用的数据一般都放在栈内存中 存储方式:先入后出 后入先出
  • 引用类型的堆栈内存:栈内存存储的是引用类型的地址 堆内存存储的是真实的数据, Object 和 Array 占用空间大,大小不固定,存储方式:先入先出 后入后出

基本类型(值类型): Number ,String, Null ,Boolean, Undefined ,Symbol(es6新增 独一无二的值) ,Bigint(ES10新增 大整型)

function deepClone(o){
    var result;
    if(o instanceof Array) {  //判断是不是数组
        result = []; //是数组则赋为数组类型
    }else if(o instanceof Object) {  //判断是不是对象
        result = {}; //是对象则赋为对象类型
    }else {  //不是数组和对象就是基本类型
        return o; //直接返回
    } 
    for(let i in o){ //遍历数组或对象
        result[i] = deepClone(o[i]);
    }
    return result;
}

# 原型

参考 对象:原型法创建对象,我们可以通过以下方式重写Array对象中的任意方法,参考数组去重食用

  • 在每一个对象上都有一个prototype属性,这个属性是专门用于保存所有该对象下实例所共享的属性和方法,这个prototype属性就是我们所谓的原型对象
Array.prototype.push = function(item){
    this[this.length] = item;
}
Array.prototype.pop = function(){
    this.length = this.length - 1;
}
  • 在每一个实例上都有一个__proto__属性,用于查找该实例的原型对象
var arr = new Array();
console.log(Array.prototype); // Array []
console.log(arr.__proto__); // Array []

代码规范:使用class,避免直接操作 prototype

// bad
function Queue (contents = []) {
  this._queue = [..contents]
}
Queue.prototype.pop = function () {
  const value = this._queue[0]
  this._queue.splice(0, 1)
  return value
}

// good
class Queue {
  constructor (contents = []) {
    this._queue = [...contents]
  }

  pop () {
    const value = this._queue[0]
    this._queue.splice(0, 1)
    return value
  }
}

# 原型链

在每一个对象上都有一个__proto__的属性,该属性指向所在的原型,当对象上调用某个没有的方法时,就会通过__proto__去找原型对象查看,如果原型对象也没有该方法,则继续通过__proto__向上查找,一直找到null为止,这种层层查找的方式,被称之为原型链

# Arguments对象

arguments是一个对应于传递给函数的参数的类数组对象.简单说就是保存实参的集合

类数组

类数组意味着arguments有length属性,并且属性的索引是从零开始的,但是它没有Array的内置方法.例如forEach()和map()都是没有的

arguments对象是所有(非箭头)函数中都可用的局部变量,你可以使用arguments对象在函数中引用函数的参数,此对象包含传递给函数的每个参数,第一个参数在索引0处

function max() {
    var m = 0;
    for (let i = 0; i < arguments.length; i++) {
        if (arguments[i] >= m) {
            m = arguments[i]; // 引用实参
        }
    }
    return m;
}

var maxVal = max(1, 4, 6, 9, 10);
console.log("maxVal:", maxVal);  // maxVal: 10




 







var arr = [1, 4, 6, 9, 10]
function max(arr) {
    var m = 0;
    for (let i = 0; i < arr.length; i++) {
        if (arr[i] >= m) {
            m = arr[i];
        }
    }
    return m;
}

var maxVal = max(arr);
console.log("maxVal:", maxVal);

# 内存

栈(Stack)

栈是限定仅在表尾进行插入或删除操作的线性表.对于栈来说表尾具有特殊的含义,称为栈顶(top),表头端称为栈底(bottom),不含元素的空表称为空栈(top = bottom / top = 0).栈的修改按照后进先出的原则,因此又称为后进先出(Last in first out)的线性表

基本数据类型存放在栈内存中

var a = 10;
var b = a;

a = 30;

console.log("a =", a, "b =", b);  // a = 30 b = 10

引用数据类型存放在堆内存中,变量引用其内存地址(就是个指针)

var c = [1];
var d = c;

c[0] = 2;

console.log("c =", c, "d =", d)
// c = Array [ 2 ] d = Array [ 2 ]

以上例子c,d都是引用类型,栈内存中存放地址指向堆内存中的数组,引用类型的复制会为新的变量自动分配一个新的值保存在变量中,但只是引用类型的一个指针,它们指向同一个数组

console.log([] === [])  // false
console.log({} === {})  // false

# 时间循环机制

所有的任务队列可以分为同步任务和异步任务.同步任务会进入主线程中执行,异步任务回通过任务队列（Event Queue）的机制进行协调执行

# 同步异步

同步和异步任务分别进入不同的执行'场所',同步进入主线程,异步进入Event Table并注册函数.当指定的事情完成时.Event Table会将这个函数移入任务队列（Event Queue).主线程内的任务执行完毕为空,就去任务队列（Event Queue）读取对应的函数.进入主线程执行

console.log('11111');
for(var a = 1; a < 5; a++){
    setTimeout(() => {
        console.log('a:', a);
    }, 0);
    console.log(a);
}
console.log('22222');
// 11111
// 1
// 2
// 3
// 4
// 22222
// a: 5
// a: 5
// a: 5
// a: 5
