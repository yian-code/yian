# 进程与线程

# 进程

进程就是正在运行的程序，它会占用对应的内存区域，由CPU进行执行与计算，进程是操作系统分配资源的单位

  • 独立性

进程是系统中独立存在的实体，它可以拥有自己独立的资源，每个进程都拥有自己私有的地址空间，在没有经过进程本身允许的情况下，一个用户进程不可以直接访问其他进程的地址空间

  • 动态性

进程与程序的区别在于，程序只是一个静态的指令集合，而进程是一个正在系统中活动的指令集合，程序加入了时间的概念以后，称为进程，具有自己的生命周期和各种不同的状态，这些概念都是程序所不具备的.

  • 并发性

多个进程可以在单个处理器CPU上并发执行，多个进程之间不会互相影响.

# 线程

线程是操作系统OS能够进行运算调度的最小单位，它被包含在进程之中，是进程中的实际运作单位，线程的本质就是函数执行.

  • 一个进程可以开启多个线程，其中有一个主线程来调用本进程中的其他线程.
  • 进程的切换，切换的是不同进程的主线程
  • 多线程可以让同一个进程同时并发处理多个任务，相当于扩展了进程的功能，了解高并发问题.

线程的状态: 就绪、执行、阻塞，或者分为创建、就绪、执行、阻塞、终止(死亡)

# JS-AJAX

# 请求方式

  1. GET:  通过url查询字符串进行传递，不太安全
    • 具有缓存性
    • 传输效率高
    • 可以传递的数据量小，受url长度限制
  2. POST:  传参方式是在send请求体中传递，较安全
    • 没有缓存性
    • 传输效率没GET高
    • POST没有数据量限制

# 前端(GET)

  1. open规定请求的类型，URL以及是否异步处理请求
  • 语法:open(method,url,async)
    • method: 请求类型GET或POST
    • url: 文件在服务器上的位置
    • async: true(异步),false(同步)
  1. send将请求发送到服务器

send(string) : string仅用于POST请求

// 1. 创建xml对象实例
let xml = new XMLHttpRequest();
// 2. 打开与后台的通道
xml.open('GET', 'http://localhost:5000/add?userName=' 
                + ipts[0].value + '&password=' + ipts[1].value);
// 3. 发送请求链接
xml.send(); 
// 4. 监听服务器响应状态
xml.onreadystatechange = function () {
    if(xml.status === 200 && xml.readyState === 4) {
        // 服务器响应返回数据 xml.response
        console.log(JSON.parse(xml.response));
    }
}
  1. onreadystatechange事件每当 readyState 属性改变时,就会调用该函数
  • status:   http状态码
    • 1xx:  http请求中
    • 2xx:  响应完成并成功返回  200
    • 3xx:  响应完成,资源重定向 301,302
    • 4xx:  响应发生客户端错误 404
    • 5xx:  响应发生服务端错误 (有可能传参错误)
// 监听服务器响应状态
xml.onreadystatechange = function () {
    if(xml.readyState === 4) {
        if(xml.status >= 200 && xml.status < 300) {
            console.log(xml.response); // 服务器响应回来的数据
        }
    }
}
  • readyState:  存有 XMLHttpRequest 的状态,从 0 到 4 发生变化.
    • 0: 请求未初始化
    • 1: 服务器连接已建立
    • 2: 请求已接收
    • 3: 请求处理中
    • 4: 请求已完成,且响应已就绪

# 跨域问题

同源策略(Same-origin policy)

协议、域名(IP)、端口相同即为同源.浏览器的同源策略是一种约定,是浏览器最核心也是最基本的安全功能.如果缺少了同源策略,浏览器很容易受到XSS、CSFR 等攻击.

由于同源策略只允许同协议,同域名,同端口号的网址可以相互访问,三者有一个不同即为跨域,解决跨域问题的方式:(一般用前三个)

  • CORS解决跨域(后端解决)

    res.setHeader('Access-Control-Allow-Origin', '*');
    next();

app.use(bodyParser.json())

    // console.log(req.query);   这个接收方式只适用于get

  • post中逻辑文件setRegister.js
const fs = require('fs');

function setRegister(req, res) {
    let userObj = req.body;
    // 读用户注册文件,判断用户是否已经注册
    fs.readFile('./user.json', 'utf-8', function(err, data) {
        let arr = JSON.parse(data);
        let flag = true;
        arr.forEach( function (item) {
            if (item.userName === userObj.userName) {
                flag = false;
                res.send({
                    code: 0,
                    msg: '该用户已存在',
                    data: ''
                })
            }
        })
        // 注册用户
        if(flag) {
            arr.push(userObj);
            fs.writeFile('./user.json', JSON.stringify(arr), function(err) {
                if (err) throw err;
                res.send({
                    code: 1,
                    msg: '注册成功',
                    data: ''

                })
            })
        }
    })
}
exports.setRegister = setRegister;
