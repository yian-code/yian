# 图片的预加载和懒加载

前端性能优化之图片

  • 精灵图(雪碧图) 将图片转换为base64字符串格式使用 使用svg图标
  • 懒加载 : 页面一加载的时候,不会将所有的图片都加载完毕,而是当用户需要查看该图片时,再进行加载
  • 预加载 : 将图片从服务器提前获取到,然后存入本地缓存中,之后用户查看图片的时候,就不需要从服务器获取,而是直接从本地缓存中读取,这样读取的速度会更快,更方便轮播图可能会使用
// 预加载
var imgUrl = ["./imgs/1.png","./imgs/2.png","./imgs/3.png","./imgs/4.png"]
var img = new Image();
console.log(img)
img.src = imgUrl[0];
img.onload = function(){	
    // 图片加载完再执行的操作
}

// 先加载可视页面的图片

# 同步异步

需求 ：0 1 2 3 4

//闭包
for (var i = 0; i < 5; i++) {
    (function(i) {
        setTimeout(function() {
            console.log(i); //5个值
            //内部 没有i 上一层作用域找--->函数a--->有参数i -->使用i
        })
    })(i);
}

//使用let 块级作用域解决
for (let i = 0; i < 5; i++) {
    setTimeout(function() {
        console.log(i); //5个值
    })
}

# 多维数组降维(递归)

// 接收降维后的数组
var newArr = [];

// 递归函数
function fn(arr) {
    arr.forEach(item => {
        if(Array.isArray(item)) {
            fn(item);
            // arguments.callee(item);  // 与fn(item)等同
        } else {
            newArr.push(item);
        }
    });
    return newArr;
}
let  arr = [1, 2, [3, 4, [5, 6, [7, 8, [9] ] ] ] ]

console.log(fn(arr))
// [1, 2, 3, 4, 5, 6, 7, 8, 9]

# display:none和visibility:hidden区别​

  • visibility可以隐藏元素,但是元素仍然占用未隐藏时候的空间,元素虽然被隐藏了,但是影响布局
  • display:none隐藏元素且不会占用空间,不影响布局

# 数组去重

ES6 提供了新的数据结构 Set。它类似于数组，但是成员的值都是唯一的，没有重复的值

let arr = [1, 3, 2, 1, 1, 4, 3, 6, 0, 4];
let newArr = [...new Set(arr)];

console.log(newArr);
// Array(6) [ 1, 3, 2, 4, 6, 0 ]

# web认识与理解

WEB标准不是某一个标准，而是一系列标准的集合。网页主要由三部分组成：

  1. 结构（Structure）结构化标准语言主要包括XHTML和XML
  2. 表现（Presentation）表现标准语言主要包括CSS
  3. 行为（Behavior）行为标准主要包括对象模型（如W3C DOM）、ECMAScript并要求这三部分分离。

这些标准大部分由W3C起草和发布，也有一些是其他标准组织制订的标准，比如ECMA（European Computer Manufacturers Association）的ECMAScript标准。

# 行内元素与块级元素

  • CSS规范规定，每个元素都有display属性，确定该元素的类型，每个元素都有默认的display值，比如div 默认display属性值为“block”，成为“块级”元素；span默认display属性值为“inline”，是“行内”元素。
    • 行内元素有：a 、b、 span、img、 input、 select、 strong（强调的语气）
    • 块级元素有：div、 ul、 ol、 li、 dl、 dt、 dd、 h1、 h2、 h3、 h4…p

# npm 与 yarn

yarn是为了弥补npm的缺陷而生的

npm的缺陷：

  1. npm 下载很慢
  2. 同一个项目，安装时无法保证包的一致性
"^3.2.37"
"~3.2.37"
"3.2.37"

yarn的优点：

  1. 并行安装，速度快
  2. 离线模式，如果之前已经安装过的包，则会从缓存中获取，不像npm从网络中下载
  3. 安装版本统一
  4. 更简洁的输出，只会打印必要的信息
  5. 更好的语义化

npm与yarn命令对比：

npm install        
yarn

npm install react--save 
yarn add react

npm uninstall react--save 
yarn remove react

npm install react --save-dev
yarn add react --dev

npm update --save 
yarn upgrade
