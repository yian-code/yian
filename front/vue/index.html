# 计算属性和侦听器

计算属性computed: 调用计算属性时不需要(),它是属性不是方法.计算属性在所依赖的变量不发生改变的时候不会执行,多次使用时只执行一次,其他结果是取得的缓存下来的结果

<div id="app"> {{ reverseStr }} </div>

<script>
const app = new Vue({
    el: '#app',
    data: { 
        str: 'hello world!'
    },
    computed: {
        reverseStr() {
            return this.str.split('').reverse().join('');
        },
    }
})
</script>

侦听器watch: newStr:改变后的数据.oldStr:改变前的数据

const app = new Vue({
    el: '#app',
    data: { 
        str: 'hello world!'
    },
    watch: {
        str: function (newStr, oldStr) {
            console.log('newStr:', newStr, 'oldStr:', oldStr);
        }
    }
})

# render函数

字符串模板的代替方案,允许你发挥 JavaScript 最大的编程能力.该渲染函数接收一个 createElement 方法作为第一个参数用来创建 VNode

const app = new Vue({
    render(createElement) {
        return createElement('div', {
            class: {
                'foo': true,
                'foz': false 
            },
            style: {
                'color': 'red',
                'fontSize': '30px'
            }
        },[
            '我是render创建的内容',
            createElement('a', { // 在创建的DOM中继续创建
                'class': 'fob',
                'style': {
                    'textDecoration': 'none',
                    'color': '#333'
                },
                attrs: {
                    'href': 'https://www.baidu.com'
                }
            }, '百度')
        ])
    }
}).$mount('#app')

# vm.$nextTick

在下次 DOM 更新循环结束之后执行延迟回调.在修改数据之后立即使用这个方法,获取更新后的 DOM

<div id="app">
    <button type="button" @click="fn">添加li</button>
    <!-- 获取DOM时候需要使用$refs -->
    <ul ref="ul"> 
        <li v-for="item in arr">{{ item }}</li>
    </ul>
</div>
<script>
    const app = new Vue({
        data: {
            arr: ['aaa', 'bbb', 'ccc', 'ddd']
        },
        methods: {
            fn() {
                this.arr.push('eee');
                /** 直接获取的话会发现childNodes与实际不符
                 * let uls = this.$refs.ul;      获取DOM   $refs
                 * console.log(uls.childNodes);  获取ul的子节点li
                **/
                this.$nextTick( () => {  // 异步跟新队列,让DOM重新加载后获取DOM元素
                    let uls = this.$refs.ul;     
                    console.log(uls.childNodes); 
                })
            }
        },
    }).$mount('#app')
</script>

# vm.$refs

一个对象,持有注册过 ref attribute 的所有 DOM 元素和组件实例

<div id="app">
    <p ref="p">{{ msg }}</p>
    <button type="button" ref="btn" @click="fn">获取</button>
</div>
<script>
    const app = new Vue({
        data: {
            msg: '默认值'
        },
        methods: {
            // let p1 = this.$refs['p'];
            fn() { let p = this.$refs.p; }    
        },
    }).$mount('#app');
    app.msg = '修改msg';
    console.log('p.innerHTML', app.$refs.p.innerHTML); // p.innerHTML 默认值
    Vue.nextTick( () => {
        console.log('p.innerHTML1', app.$refs.p.innerHTML); // p.innerHTML1 修改msg
    })
</script>

组件上的refs不仅可以获取组件的数据,还可以调用组件的方法

<div id="app">
    <yi-an ref="ya"></yi-an> <!-- 绑定在组件上 -->
    <button type="button" @click="add">获取</button>
</div>
<script>
    const YiAn = {
        template: `<div @click="fn">{{ msg }}</div>`,
        data() { 
            return { 
                msg: '组件内容' 
            } 
        },
        methods: {
            fn() { console.log('fn函数执行') }
        }
    }
    const app = new Vue({
        data: { },
        methods: {
            add() {
                let ya = this.$refs['ya'];
                let data = this.$refs['ya'].msg; // 获取组件的数据
                this.$refs['ya'].fn(); // 调用组件的方法
            }
        },
        components: { YiAn }
    }).$mount('#app');
</script>

# 父子组件

全局与局部、父子组件open in new window在学习父子组件之间的数据传递的时候，出现了这样一个问题:

  1. 我先定义了一个全局组件Father
  2. 又写了一个局部组件Son
  • 当以局部组件Son组件作为父组件调用Father全局自定义组件作为子组件时一切正常
  • 当我以全局组件Father作为父组件调用局部自定义组件Son作为子组件时,发现无法做到

原因

局部组件只能在被引用的实例中使用,所以需要在相应实例中加入component的组件声明,声明里边要引入template所在的变量名

  • 解决问题: 在全局自定义组件内部加入component的组件声明
// <Father></Father>

const Son = {
    data() {
        return {
            msg: '我是子组件'
        }
    },
    props: ['msgs'],
    template: `<div> {{ msg }}  {{ msgs }} </div>`
}
// 全局自定义组件
Vue.component('Father', {
    // 组件中的data一定是函数，返回的是对象
    data: function() {
        return {
            num: 'hello vue component',
            test: '传递给子组件的数据'
        }
    },
    template:`<div> {{ num }} <Son v-bind:msgs='test'></Son> </div>`,
    components: {
        Son
    }
})








 








 


 
 



父传子

  • 在父组件中的子组件标签上通过自定义属性传递需要传递给子组件的数据
  • 在子组件内部,通过props接收父组件传递给子组件的自定义属性,这个属性就是父组件传递过来的数据
<!-- <Father></Father> -->

<script>
    // 父组件通过自定义事件绑定getSon函数
    let Father = {
        data() {
            return {
                msg: '我是Father',
                msg2: ''
            }
        },
        template:`
            <div>
                <h3> {{ msg }} {{ msg2  == '' ? '': '---' + msg2 }}</h3>
                <Son @getSon="getVal"></Son>
            </div>
        `,
        methods: {
            // 定义了一个getVal事件，将Son传来的值送入msg2，渲染到页面上
            getVal(val) {
                console.log('val: ', val);
                this.msg2 = val;
                console.log(this.msg2)
            }
        }
    }
    // toFather函数通过this.$emit触发getSon事件
    Vue.component('Son',{
        data() {
            return {
                msg1: '我是Son',
                val: ''
            }
        },
        template:`
            <div>   
                <h2>{{ msg1 }}</h2>
                <input type="text" v-model="val">
                <button type="button" @click="toFather">传值按钮</button>
            </div>
        `,
        methods: {
            // 定义了一个按钮事件，触发传值函数getSon
            toFather() {    
                // console.log(this.val);
                this.$emit('getSon', this.val);
                this.val = '';
            }
        },
    })
    const app = new Vue({
        el: '#app',
        components: {
            Father,
        }
    })
</script>

子传父

// Son
//   --> toFather         
//      --> this.$emit('getSon', this.val) 
// Father
//   --> @getSon="getVal" 
//      --> getVal 
//         --> this.msg2 = val

# 动态组件&异步组件

当在这些组件之间切换的时候,你有时会想保持这些组件的状态,以避免反复重新渲染导致的性能问题,我们可以使用keep-alive来解决

<!-- 四个组件的name: aw  bw cw dw  -->
<!-- 无缓存 -->
<component :is="comps"></component>
<!-- 有缓存的 -->
<keep-alive> 
    <component :is="comps"></component>
</keep-alive>
<!-- include :   字符串或正则表达式。只有名称匹配的组件会被缓存 -->
<keep-alive include="aw,bw"> 
    <component :is="comps"></component>
</keep-alive>
<!-- exclude :  与include相反,被匹配到的不缓存 -->
<keep-alive exclude="aw,bw"> 
    <component :is="comps"></component>
</keep-alive>
<!-- max : 最多缓存几个 -->
<keep-alive :max="4"> 
    <component :is="comps"></component>
</keep-alive>

动态组件的切换中有连个生命周期

activated() {
    console.log('ComYi组件被激活')
},
deactivated() {
    console.log('ComYi组价失活')

# Vue.filter&filters

过滤器表达式{{ 变量 | 过滤器名称 }},过滤器前面的变量会作为参数传递给过滤器函数,过滤器函数必须要有返回值

# 局部过滤器

<div id="app">
    <!--过滤器应该被添加在 JavaScript 表达式的尾部，由“管道”符号指示-->
    {{ time | timeFormat }} 
</div>
<script>
    const app = new Vue({
        data: { time: +new Date() },
        filters: { //过滤器
            timeFormat(val){ // 格式化时间函数
                let t = new Date(val);
                let y = t.getFullYear() + '';
                let m = t.getMonth() + 1 + '';
                let d = t.getDate() + '';
                let h = t.getHours() + '';
                let mm = t.getMinutes() + '';
                let ss = t.getSeconds() + '';
                return `${y}/${m.padStart(2, '0')}/${d} ${h}/${mm}/${ss}`;
            }
        }  
    }).$mount('#app');

Vue.filter('timeFormat', (val) => {
    let t = new Date(val);
    let y = t.getFullYear() + '';
    let m = t.getMonth() + 1 + '';
    let d = t.getDate() + '';
    let h = t.getHours() + '';
    let mm = t.getMinutes() + '';
    let ss = t.getSeconds() + '';
    return `${y}/${m.padStart(2, '0')}/${d} ${h}/${mm}/${ss}`;
})

# Vuex

Vuexopen in new window 是一个专为 Vue.js 应用程序开发的状态管理模式.它采用集中式存储管理应用的所有组件的状态,并以相应的规则保证状态以一种可预测的方式发生变化

// <add />

const store = new Vuex.Store({  // 创建仓库
    state: { count: 0 },        // 数据源
    getter: {                   // 分发数据
        getCount(state) { return state.count; }
    },
    // 同步更改 Vuex 的 store 中的状态的唯一方法是提交 mutation
    mutations: {                
        addmutations(state) {   // 参数不止state
            state.count++;
        }
    },
    // 异步触发mutations来修改状态
    actions: {
    // Action函数接受一个与 store 实例具有相同方法和属性的context对象
        addmutations (context) {
            context.commit('addmutations')
        }
    }
});

const Test = {
    // .....
    methods: {
        add(){
        // 要唤醒一个 mutation 处理函数，你需要以相应的 type 调用 store.commit 方法
            // this.$store.commit('addmutations') 
        // Action 通过 store.dispatch 方法触发
            this.$store.dispatch('addmutations')
        }
    }
}

const app = new Vue({
    store,                      // 使用vuex
}).$mount('#app')
